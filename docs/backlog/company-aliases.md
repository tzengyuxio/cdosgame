# 公司別名 / 改名 → redirect 候選

## 觀念釐清（重要）

兩種情況要分開看：

- **純別名**（同一家公司、只是名字寫法不同：簡繁、全名/簡稱、中英）——已由各公司 md 的
  `aliases` 欄位涵蓋（供搜尋/辨識）。**通常不需要 redirect**，因為不會有人去連舊寫法的網址。
- **改名 / 先後傳承**（兩個都是正式名、有承接關係，但不想開兩頁、想保留關係）——這才用
  redirect 合成一頁。範例：**世紀縱橫 → 貳碼科技**（已啟用）。

→ redirect 精選表 `src/lib/company-aliases.js` **只收第二類**。加一筆 = redirect + 別名發行的
遊戲自動歸 canonical 頁（兩處同步）。

## 注意

- 精選表**不要**塞會撞到既有公司頁 id 的別名（會蓋掉那頁），程式無額外防呆。
- 同一別名只能指一個 canonical。
